# Oyez Arguments

A simple, mobile-first **website** for listening to U.S. Supreme Court **oral arguments** on
a phone — a friendlier front-end over [Oyez](https://www.oyez.org), whose own mobile
experience is weak. Personal, non-commercial use.

**▶ Live site: https://chicoboy700.github.io/Oyez-Arguments/**
(served by GitHub Pages from the [`docs/`](docs/) folder — see [DEPLOY.md](DEPLOY.md)).

## What it does
- Browse by **term** (year) → that term's argued cases → a **player**.
- **Search by advocate** (e.g. "Paul Clement") across every term, tap straight to the audio.
- Audio **streams live** from Oyez (never downloaded/re-hosted), with lock-screen controls
  (Media Session API) and **installable PWA** support for background playback on a phone.

## How it's built
- Plain HTML/CSS/JS, no framework. The whole site is in [`docs/`](docs/).
- Browsing is live against the public Oyez API (`api.oyez.org`, which allows direct browser
  calls). Oyez has no working "cases by advocate" lookup, so advocate search is powered by a
  pre-built index, [`docs/data/advocates.json`](docs/data/advocates.json), generated by
  [`tools/build_advocate_index.py`](tools/build_advocate_index.py).
- The full plain-English build story is in [BUILD_LOG.md](BUILD_LOG.md), with a polished
  Word version in `HOW_I_BUILT_THIS.docx`.
